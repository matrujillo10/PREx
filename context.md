# Context — PREx project

Conversation summary up to 2026-05-09.

## Premise

AI now writes most code in many orgs. Human review is the bottleneck. Project goal: rethink the PR review experience around the premise that humans are AI code reviewers as a key piece of their job.

Backed by hard data:
- DORA 2025: review time +441%, no-review merges +31%, incidents per PR +242.7%.
- 17M agent PRs/month on GitHub (Mar 2026), +325% in 6 months.
- CodeRabbit 470-PR study: AI PRs have 75% more logic errors, 2.74x security issues, 3x readability issues, 8x more I/O ops.
- Anthropic internal: substantive PR review went from 16% → 54% after multi-agent reviewer shipped.

## Two reviewer personas (core design choice)

1. **AI-assisted author** doing pre-PR self-review.
2. **Peer reviewer** reviewing code generated by others (with AI).

Same diff primitive, different chrome around it. Same surface = wrong default — implies cognitive jobs are same, produces rubber-stamping.

Cognitive split:
- Self-review catches **intent**, fails at **what's on page** (familiarity blindness).
- Peer review catches **what's on page**, misses **intent** (no recall).
- AI-assisted author has *neither* advantage. Role inverted toward peer reviewer (Hashimoto: "if they're coding, I want to be reviewing").

## Type-aware design

PR type is a primary axis because each type radically changes the review checklist. Reviewer concerns vary sharply by type — refactors get more comments than feats, chores get rubber-stamped, etc.

**Full vision spine (10-12 types):** feat, fix, refactor, perf, test, docs, chore (folds build/ci/style), deps, security, migration, config, prompt/skill.

**Orthogonal axes (8) compose:** scope/area, size, SemVer impact, risk/blast radius, reversibility, generated-vs-handwritten, author class (human/AI/bot), breaking marker.

**MVP scope (decided):** start with **feat + fix + chore** (chore folds build + ci + style).

Why these three:
- Cover majority of PR volume.
- `chore` highest rubber-stamp class (74-92% accept) — biggest discipline leverage cheaply.
- `feat` and `fix` are highest burden-increase types.
- Defers harder-to-spec types (migration, security, prompt/skill) until manifest + checklist mechanic is validated.

## MVP wedge — single biggest unbuilt UX primitive

**Type-conditional review checklist rendered inline on diff hunks.** Driven by `.review/types.yaml` manifest binding type → globs → checklist → required reviewers → required checks → AI agent prompt.

Why this wedge:
- Classification → routing → CI checks → AI reviewer specialization layers are mature already.
- UI layer is the laggard. GitHub Files Changed renders identically for 3-line typo and 12-file schema migration.
- Manifest is platform-team owned, not author owned — kills label-gaming.

## Architecture sketch (pre-validation, full vision)

```
[author writes w/ AI]
   ↓
[adversarial subagent — different model — runs against type-specific rubric]
   ↓
[author self-review surface]
   - PR Contract (intent / proof / risk / AI sections / focus)
   - Prompt-skill diff sidebar
   - Scope-creep flag (>5 unrelated files)
   - Type detected → checklist inline on diff
   - "Explain in own words" → becomes PR description
   ↓ [refused if Contract blank]
[risk-tier router]
   trivial → auto-approve (Ona pattern)
   standard → 1 peer + AI panel
   sensitive → N peers + full AI panel + verifier
   ↓
[peer review surface]
   - Author Contract pinned top
   - Architectural context: call graph, blast radius, downstream
   - Type-conditional checklist inline on diff hunks
   - Risk-tier badge drives default depth
   - AI findings deduplicated, ranked by verifier
```

For MVP: skip risk-tier router and adversarial subagent. Build the inline type-conditional checklist + manifest first.

## Research conducted

Two rounds of parallel research agents on 2026-05-09.

Round 1 (5 agents):
1. Tools landscape (CodeRabbit, Greptile, Cloudflare, Anthropic ultrareview, etc.).
2. Pain points reviewing AI PRs (volume, slop, rubber-stamping, OSS DDoS, etc.).
3. Novel UX patterns (multi-agent panels, spec-as-contract, trace review, etc.).
4. Thought leader takes (Willison, Karpathy, Beck, Hashimoto, Cherny, Saffron, Doctorow).
5. Empirical / academic evidence (Cisco, Bacchelli, Sadowski, GitClear, DORA, Apiiro, METR).

Round 2 (4 agents):
6. PR / change-type taxonomies (Conventional Commits, Angular, gitmoji, k8s, Swanson).
7. Per-type review concerns + AI failure modes per type.
8. Self-review vs peer-review cognitive split + UX implications.
9. Type-aware tooling landscape + MVP recommendation.

Detailed extracts in `research.md`. Decisions in `decisions.md`.
