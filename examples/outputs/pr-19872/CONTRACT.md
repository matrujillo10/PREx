# PREx output contract

> **Generated by `prex review` on 2026-05-09 23:32 UTC.** Re-rendered every run.

This document is the contract between **PREx** (which produces structured
PR-review artifacts) and **the next layer** that consumes them — most often a
generative-UI host LLM that turns these structures into UI components, but
also any dashboard, bot, or automated reviewer.

## 5. Citation discipline

Every prose field that PREx emits — `ReviewBrief.headline`,
`HunkInsight.one_liner`, `ReviewPlan.steps[*].{what,why,impact}`,
`ReviewPlan.overview` — carries `cites: List[Citation]`. A `Citation`
resolves in one of four ways:

| `kind` | `ref` format | Where to look |
|---|---|---|
| `node` | `<kind>:<id>` | A node in `graph.json#/nodes`. |
| `edge` | `<type>:<source>->-<target>` | An edge in `graph.json#/edges`. |
| `file_line` | `<repo-relative-path>#L<start>-L<end>` | The source repo at `pr.head_sha`. |
| `external_doc` | absolute URL | The web. |

**The next layer must render a fact only via its citation.** Unsourced prose
in PREx output is a validation error.

## 6. Derivation + score

Both `Node` and `Edge` carry `derivation` and `score` fields:

- `derivation: Literal["ast","diff","crossref_text","llm","manifest","heuristic"]` —
  where the fact came from. Use this for the "evidence" tag.
- `score: float` (0..1) — the resolver's trust. Use this for visual weighting.

## 7. What the next layer should NOT do

1. Don't paraphrase prose without keeping the original `cites` references.
2. Don't render an `Edge` whose `derivation == "llm"` without surfacing that
   the relationship was inferred (e.g. show a "verified by Claude" / "AI inferred"
   badge). Same for any `HunkInsight.one_liner`, headline, or plan prose.
3. Don't render `diagnostics` whose `level == "info"` to the end user — they
   are plumbing.
4. Don't sort `plan.steps` by anything other than `rank`. The order is the
   recommended reading order.

## 8. Streaming order

When PREx adds streaming output (forthcoming), the emit order will be:

1. `pr` (PRMetadata)
2. `review.headline` (≤140 chars; first useful render)
3. `review` (full ReviewBrief)
4. `plan.overview`
5. `plan.steps[]` (top-down by rank)
6. `hunks[]`
7. `checklist[]`
8. `diagnostics[]`

The host LLM can begin rendering as soon as each block lands.
